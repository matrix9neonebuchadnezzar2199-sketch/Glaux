# Glaux Decision Log

## D-001: Tauri 不採用、Rust + eframe/egui 採用

- **日付**: 2026-06-30
- **判断**: 厳密な単一 EXE 配布では Tauri の sidecar 配置が複雑になるため、ネイティブ Rust UI を採用
- **代替案**: Tauri + externalBin（ポータブルフォルダ配布向け）

## D-002: Gemma 4 E2B 標準同梱

- **日付**: 2026-06-30
- **判断**: v1 の標準モデルは Gemma 4 E2B IT GGUF Q4 系。context 2048、同時推論 1 本
- **根拠**: [Gemma 4 モデルカード](https://ai.google.dev/gemma/docs/core/model_card_4?hl=ja)

## D-003: 自己展開ランタイム

- **日付**: 2026-06-30
- **判断**: EXE 内に `llama-server` と GGUF を埋め込み、初回起動時に `%LOCALAPPDATA%/Glaux/runtime/<hash>/` へ展開
- **理由**: 単一配布物と実行時 mmap の複雑さを避ける

## D-004: チャット履歴非保存

- **日付**: 2026-06-30
- **判断**: 会話はメモリ上のみ。ファイル・DB 永続化なし
- **UI**: Help とコンテキスト概算 + クリーンで明示

## D-005: モデル解説はオフライン同梱テキスト

- **日付**: 2026-06-30
- **判断**: 外部リンクなし。`assets/texts/model-intro.md` を同梱表示
- **注意**: 未検証の強い数値断定はアプリ内本文では避ける

## D-006: ポータブルフォルダ配布へ移行（v0.2）

- **日付**: 2026-07-01
- **判断**: 単一 EXE 埋め込みを廃止し、`Glaux.exe` + `runtime/` + `model/` のフォルダ配布に変更
- **理由**: DLL 同梱の確実性、配布サイズの透明性、AV/初回展開問題の回避
- **追加**: E2B を同梱（当初 E4B も検討したが重量のため不採用）

## D-009: E4B 不採用（v0.2）

- **日付**: 2026-07-01
- **判断**: `gemma-4-E4B-it-qat-UD-Q4_K_XL` の同梱・選択 UI を削除
- **理由**: 配布サイズ・メモリ要件が大きく、Glaux の軽量オフライン用途と合わない

## D-010: E2B を Unsloth QAT Q2 mobile に変更

- **日付**: 2026-07-01
- **判断**: 標準 E2B を `gemma-4-E2B-it-qat-UD-Q2_K_XL.gguf`（約 2.2GB）に差し替え
- **理由**: 4GB 級端末で Q4（約 3.3GB）がメモリ不足で llama-server が AV 終了する事例あり。QAT Q2 で RAM 要件を下げる
- **互換**: 旧 `gemma-4-e2b-it.gguf` は `from_filename` で E2B として認識可能

## D-007: 外観プリセット厳選

- **日付**: 2026-07-01
- **判断**: テーマは Glaux（ダーク）/ ライトモードの 2 種類のみ
- **理由**: UI 一貫性とライト時の文字色制御を確実にする

## D-008: コンテキスト長 15000

- **日付**: 2026-07-01
- **判断**: 既定 context_length を 15000 に引き上げ
- **注意**: 実効コンテキストは端末メモリとモデル性能に依存。概算警告は従来通り
